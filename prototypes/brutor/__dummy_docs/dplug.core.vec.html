  • alignedMalloc

    Declaration

    nothrow @nogc void* alignedMalloc(size_t size, size_t alignment);

    Allocates an aligned memory chunk. Functionally equivalent to Visual C++ aligned_malloc. Do not mix allocations with different alignment.

    Important: alignedMalloc(0) does not necessarily return null, and its result has_ to be freed with alignedFree.

    Important: Don't call that often in a real-time thread without amortization (keep a capacity). If you need to allocate from nextBuffer, do prefer the use of Vec which doesn't shrink, and will reuse the allocation.

  • alignedFree

    Declaration

    nothrow @nogc void alignedFree(void* aligned, size_t alignment);

    Frees aligned memory allocated by alignedMalloc or alignedRealloc. Functionally equivalent to Visual C++ aligned_free. Do not mix allocations with different alignment.

  • alignedRealloc

    Declaration

    nothrow @nogc void* alignedRealloc(void* aligned, size_t size, size_t alignment);
    nothrow @nogc void* alignedReallocDiscard(void* aligned, size_t size, size_t alignment);

    Reallocates an aligned memory chunk allocated by alignedMalloc or alignedRealloc. Functionally equivalent to Visual C++ _aligned_realloc. Do not mix allocations with different alignment.

    Discussion

    The "discard" version doesn't preserve data.

    Important: alignedRealloc(p, 0) does not necessarily return null, and its result has_ to be freed with alignedFree.

    Important: Don't call that often in a real-time thread without amortization (keep a capacity). If you need to allocate from nextBuffer, do prefer the use of Vec which doesn't shrink, and will reuse the allocation.

  • reallocBuffer

    Declaration

    nothrow @nogc void reallocBuffer(T)(ref T[] buffer, size_t length, int alignment = 1);

    Used throughout dplug:dsp to avoid reliance on GC.

    Important: Size 0 is special-case to free the slice. This works a bit like alignedRealloc except with slices as input. You MUST use consistent alignement thoughout the lifetime of this buffer.

    Parameters

    T[] buffer

    Existing allocated buffer. Can be null. Input slice length is not considered.

    size_t length

    Desired slice length.

    int alignment

    Alignement if the slice has allocation requirements, 1 else. Must match for deallocation.

    Example: 

    1. import std.stdio;

struct MyDSP
{
nothrow @nogc:

    void initialize(int maxFrames)
    {
        // mybuf points to maxFrames frames
        mybuf.reallocBuffer(maxFrames);
    }

    ~this()
    {
        // If you don't free the buffer, it will leak.
        mybuf.reallocBuffer(0);
    }

private:
    float[] mybuf;
}

    Important: Don't call that often in a real-time thread without amortization (keep a capacity). If you need to allocate from nextBuffer, do prefer the use of Vec which doesn't shrink, and will reuse the allocation. This, instead, will shrink the allocation which makes it more expensive.

  • Vec

    Declaration

    struct Vec(T);

    Kind of a std::vector replacement. Grow-only array, points to a (optionally aligned) memory location. This can also work as an output range. Vec is designed to work even when uninitialized, without makeVec.

    Warning: it is pretty barebones, doesn't respect T.init or call destructors. When used in a GC program, GC roots won't be registered.

    • allocArray

      Declaration

      void allocArray(T)(out T[] array, size_t numElems, size_t alignment = 1);
      void alloc(T)(out T* array, size_t numElems, size_t alignment = 1);

      Allocate (or count) space needed for numElems elements of type T with given alignment. This gets called twice for each array, see example for usage.

      Discussion

      This bumps the internal bump allocator. Giving null to this chain and converting the result to size_t give the total needed size for the merged allocation.

      Warning:

      • If called after a start() call, the area returned are wrong and are only for counting needed bytes. Don't use those.
      • If called after an allocate() call, the area returned are an actual merged allocation (if the same calls are done).

      Warning: Prefer allocArray over alloc variant, since the extra length field WILL help you catch memory errors before release. Else it is very common to miss buffer overflows in samplerate changes.

    • allocate

      Declaration

      nothrow @nogc void allocate();

      Allocate actual storage for the merged allocation. From there, you need to define exactly the same area with alloc and allocArray. This time they will get a proper value.
