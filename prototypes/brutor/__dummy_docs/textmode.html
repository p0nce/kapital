  • TM_CharData

    Declaration

    struct TM_CharData;

    An individual cell of text-mode buffer.

    Discussion

    Either you access it or use the print functions.

    The first four bytes are a Unicode codepoint, conflated with a grapheme and font "glyph". There is one font, and it's 8x8. Not all codepoints exist in that font.

    Next 4-bit is foreground color in a 16 color palette. Next 4-bit is background color in a 16 color palette.

      Low nibble = foreground color (0 to 15) High nibble = background color (0 to 15)

  • TM_Console

    Declaration

    struct TM_Console;

    Main API of the text-mode library.

    Discussion

    3 mandatory calls:

    TM_Console console; console.size(columns, rows); console.outbuf(buf.ptr, buf.w, buf.h, buf.pitch); console.render();

    All calls can be mix-and-matched largely without order, with everything relevant being cached.

    Note: None of TM_Console functions are thread-safe.

    • Either call them in a single-threaded way,
    • or synchronize externally.
    • or use different TM_Console objects in the first place.

      Declaration

      pure nothrow @nogc @safe void cprint(const(char)[] s);

      Print text to console at current cursor position, encoded in the CCL language (much the same as in console-colors DUB package). Text input MUST be UTF-8.

      Discussion

      Accepted tags:

      • such as:
      each corresponding to color 0 to 15 in the palette.

      Unknown tags have no effect and are removed. Tags CAN'T have attributes. Here, CCL is modified (vs console-colors) to be ALWAYS VALID.
      • STYLE tags, such as:
      , , , ,

      Escaping:

      • To pass '<' as text and not a tag, use <
      • To pass '>' as text and not a tag, use >
      • To pass '&' as text not an entity, use &

      See Also

      print
